## [◆ ](#a45b22c6c2252e2afa28ee8b1bfed3ecf)subevent\_done\_status

| enum [bt\_conn\_le\_cs\_subevent\_done\_status](group__bt__conn.md#ga8e54a191b889b1d5ed3e1fbb648d9bc7) bt\_conn\_le\_cs\_subevent\_result::subevent\_done\_status |
| --- |

Subevent status.

For aborted subevents, this will be set to [BT\_CONN\_LE\_CS\_SUBEVENT\_ABORTED](group__bt__conn.md#gga8e54a191b889b1d5ed3e1fbb648d9bc7a5943689e3233bc87054dbd9f54042202 "BT_CONN_LE_CS_SUBEVENT_ABORTED") and abort\_step will contain the step number on which the subevent was aborted. Consider the following example:

subevent\_done\_status = [BT\_CONN\_LE\_CS\_SUBEVENT\_ABORTED](group__bt__conn.md#gga8e54a191b889b1d5ed3e1fbb648d9bc7a5943689e3233bc87054dbd9f54042202 "BT_CONN_LE_CS_SUBEVENT_ABORTED") num\_steps\_reported = 160 abort\_step = 100

this would mean that steps from 0 to 99 are complete and steps from 100 to 159 are aborted.
